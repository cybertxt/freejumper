# Chrome Web Store 发布指南

本指南将帮助你将 FreeJumper 扩展发布到 Chrome Web Store。

## 准备工作

### 1. 开发者账户注册

1. 访问 [Chrome Web Store 开发者控制台](https://chrome.google.com/webstore/devconsole)
2. 使用 Google 账户登录
3. 支付一次性注册费用：**$5 USD**（一次性，永久有效）
4. 填写开发者信息（姓名、地址等）

### 2. 准备发布材料

#### 必需的图标

- **16x16** 像素 - 扩展工具栏图标
- **48x48** 像素 - 扩展管理页面图标
- **128x128** 像素 - Chrome Web Store 展示图标

> 💡 提示：使用 `icons/generate-icons.html` 生成图标，或使用专业设计工具创建

#### 商店截图/图片

准备以下图片（PNG 格式）：

1. **至少 1 张，最多 5 张截图**
   - 推荐尺寸：1280x800 或 640x400 像素
   - 展示扩展的主要功能界面
   - 建议包含：配置页面、右键菜单使用场景等

2. **小型宣传图（可选）**
   - 440x280 像素
   - 用于商店列表展示

#### 商店描述

准备以下文本内容：

1. **简短描述**（132 字符以内）
   ```
   选中文字后，通过右键菜单快速跳转到自定义URL。支持多模板配置，可设置别名，自由定制跳转目标。
   ```

2. **详细描述**（最多 132 字符用于简短描述，详细描述可更长）
   ```
   FreeJumper 是一个强大的 Chrome 扩展，允许你选中网页上的文字后，将其作为自定义URL的一部分，直接在新标签页中打开。

   主要功能：
   • 多URL模板配置 - 支持配置多个URL模板，每个模板可设置别名
   • 右键菜单选择 - 选中文字后，右键菜单中显示所有配置的跳转选项
   • 快速跳转 - 从右键菜单选择目标URL，直接在新标签页打开
   • 实时预览 - 在配置页面可以实时预览所有模板生成的URL
   • 预设模板 - 内置常用网站的URL模板，一键添加
   • 灵活管理 - 支持添加、编辑、删除模板，自由定制
   • 多语言支持 - 支持简体中文和英文，根据浏览器语言自动切换

   使用场景：
   - 快速搜索：选中文字后跳转到 Google、百度等搜索引擎
   - 代码搜索：选中代码跳转到 GitHub 搜索
   - 翻译查询：选中文字跳转到翻译网站
   - 自定义工作流：配置公司内部搜索、文档系统等

   使用方法：
   1. 安装扩展后，点击工具栏图标打开配置页面
   2. 添加或编辑URL模板（使用 {text} 作为选中文字的占位符）
   3. 在任意网页上选中文字，右键选择"FreeJumper 跳转到"
   4. 选择目标模板，自动在新标签页打开

   隐私说明：
   - 本扩展不会收集任何用户数据
   - 所有配置仅存储在本地浏览器中
   - 不会向任何服务器发送信息
   ```

3. **分类**
   - 主要类别：生产力工具（Productivity）
   - 次要类别：工具（Tools）

4. **语言**
   - 简体中文（zh_CN）
   - 英文（en）

## 打包扩展

### 1. 准备发布版本

确保所有文件都已准备好：

```bash
# 检查文件结构
freejumper/
├── manifest.json
├── background.js
├── content.js
├── content.css
├── options.html
├── options.css
├── options.js
├── popup.html
├── popup.css
├── popup.js
├── icons/
│   ├── icon16.png
│   ├── icon48.png
│   └── icon128.png
├── _locales/
│   ├── zh_CN/
│   │   └── messages.json
│   └── en/
│       └── messages.json
└── README.md
```

### 2. 创建 ZIP 文件

**重要：** 不要包含以下文件：
- `.git/` 目录
- `README.md`（可选，Chrome Web Store 不需要）
- `PUBLISHING.md`（可选）
- `.DS_Store`（macOS 系统文件）
- `node_modules/`（如果有）
- 任何测试文件

**在 macOS/Linux 上：**

```bash
# 进入项目目录
cd /Users/tomasxie/projects/freejumper

# 创建发布包（排除不需要的文件）
zip -r freejumper-v1.1.0.zip . \
  -x "*.git*" \
  -x "*.DS_Store" \
  -x "README.md" \
  -x "PUBLISHING.md" \
  -x "icons/generate-icons.html" \
  -x "icons/README.md"
```

**在 Windows 上：**

1. 选择所有需要的文件（排除上述文件）
2. 右键 → 发送到 → 压缩(zipped)文件夹
3. 重命名为 `freejumper-v1.1.0.zip`

**验证 ZIP 文件：**

- ZIP 文件大小应该小于 10MB（Chrome Web Store 限制）
- 解压后应该能正常加载为扩展

## 提交到 Chrome Web Store

### 1. 创建新项目

1. 登录 [Chrome Web Store 开发者控制台](https://chrome.google.com/webstore/devconsole)
2. 点击 **"新增项目"** 或 **"New Item"**
3. 上传 ZIP 文件
4. 等待验证完成（通常几秒钟）

### 2. 填写商店信息

#### 基本信息

- **名称**：FreeJumper
- **简短描述**：选中文字后，通过右键菜单快速跳转到自定义URL
- **详细描述**：粘贴上面准备的详细描述
- **类别**：生产力工具（Productivity）

#### 图片资源

- **图标**：上传 128x128 的图标
- **截图**：上传至少 1 张，最多 5 张截图
- **小型宣传图**（可选）：440x280 像素

#### 隐私和权限

- **隐私政策**：如果扩展不收集数据，可以填写：
  ```
  本扩展不收集、存储或传输任何用户数据。所有配置仅存储在本地浏览器中。
  ```
  
  或者创建一个隐私政策页面并填写 URL。

- **权限说明**：解释为什么需要这些权限
  - `storage`：存储用户配置
  - `activeTab`：访问当前标签页以获取选中文字
  - `tabs`：在新标签页中打开URL
  - `contextMenus`：创建右键菜单
  - `<all_urls>`：在所有网页上工作

#### 分发选项

- **可见性**：
  - **公开**：所有人可见（推荐）
  - **不公开**：仅通过链接访问
  - **私有**：仅限指定用户

- **地区**：选择要发布的地区（默认：所有地区）

- **定价**：免费

### 3. 提交审核

1. 检查所有信息是否正确
2. 点击 **"提交审核"** 或 **"Submit for Review"**
3. 等待审核（通常 1-3 个工作日）

### 4. 审核状态

- **待审核**：已提交，等待审核
- **审核中**：正在审核
- **已发布**：审核通过，已上架
- **需要修改**：审核发现问题，需要修改后重新提交

## 更新扩展

### 1. 更新版本号

在 `manifest.json` 中更新版本号：

```json
{
  "version": "1.1.1"  // 递增版本号
}
```

### 2. 创建更新包

按照上面的步骤创建新的 ZIP 文件。

### 3. 上传更新

1. 在开发者控制台找到你的扩展
2. 点击 **"上传新版本"** 或 **"Upload Updated Package"**
3. 上传新的 ZIP 文件
4. 填写 **"版本说明"**（What's new）
5. 提交审核

## 常见问题

### Q: 审核被拒绝怎么办？

A: 查看拒绝原因，通常包括：
- 权限说明不清晰
- 缺少隐私政策
- 描述不符合要求
- 功能描述不准确

根据反馈修改后重新提交。

### Q: 如何测试扩展？

A: 在提交前，使用开发者模式加载扩展进行测试：
1. 打开 `chrome://extensions/`
2. 启用"开发者模式"
3. 点击"加载已解压的扩展程序"
4. 选择项目文件夹

### Q: 版本号规则？

A: 使用语义化版本号：
- 主版本号.次版本号.修订号
- 例如：1.0.0 → 1.0.1（小修复）→ 1.1.0（新功能）→ 2.0.0（重大更新）

### Q: 可以发布到其他商店吗？

A: 可以，但需要：
- Edge Add-ons（Microsoft Edge）
- Firefox Add-ons（需要适配 Manifest V2 或 V3）
- 每个商店都有不同的要求
